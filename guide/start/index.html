系统介绍

iOpsBot 是一款面向运维工程师的"AI 运维小助手"智能化应用，不仅可以离线接入各种开源大模型，还可以额外付费线上对接 GPT API 的调用。同时还能区别于场景创建不同的应用，并通过知识投喂等方式加以训练，形成专业的客服类、主机类、网络类等场景应用库。

访问方式

👻 演示 地址：https://chat.funtech.com.cn

🤡 源码地址：http://192.168.0.100/fits-devops/iOpsBot

特性 ​

  • 🔥最新 RAG 技术 : 基于 RAG 检索增强技术，提供高效运维知识、智能问答、本地知识库等能力。
  • 🤖专属运维助手 : 随时随地服务于工程师团队，工程师可以利用它快速检索专业知识，流程化、标准化的解决故障。
  • 📦本地知识管理 : 区别于不同知识领域创建本地知识库，支持手动输入、PDF、TXT 多种数据格式进行知识投喂。
  • 🛡️私有化部署 : 安装部署简单，零配置、开箱即用，支持各大开源大模型轻松部署到本地服务器，安全可靠。
  • 👻优雅交互 : 重视 UI 交互体验，易用性。人性化的交互方式，智能便捷，快速上手。
  • 🚀API 集成 : 基于 Swagger 文档，支持标准的 API 集成，可以轻松的与现有系统进行集成。

安装 ​

环境要求 ​

  • 操作系统：Linux
  • 硬件配置：
    • 内存：32 GB
    • CPU：16 核
    • 硬盘：1 TB
    • GPU: Nvidia A100 显卡
  • 软件依赖：
    • 数据库：PostgreSQL
    • 消息队列：Redis
    • 任务调度：Celery
    • 容器编排：Docker + Docker Compose

安装步骤 ​

  1. 克隆项目
bash
git clone http://192.168.0.100/fits-devops/iOpsBot.git
  1. 修改配置文件
bash
cd iOpsBot
cp .env.example .env && vim .env
  1. 启动项目
bash
# 首次启动请构建镜像
docker-compose up --build -d

使用 ​

启动项目 ​

bash
# 启动服务
docker-compose up -d

停止项目 ​

bash
# 停止服务
docker-compose down

访问项目 ​

bash
# 访问前端页面
http://ip:3000
